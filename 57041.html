leetcode_739

发表于 2019-04-28 | 更新于 2019-04-30 | 分类于 algorithm ， LeetCode

题目：

根据每日 气温 列表，请重新生成一个列表，对应位置的输入是你需要再等待多久温度才会升高的天数。如果之后都不会升高，请输入 0 来代替。
例如，给定一个列表 temperatures = [73, 74, 75, 71, 69, 72, 76, 73]，你的输出应该是 [1, 1, 4, 2, 1, 1, 0, 0]。
提示：气温 列表长度的范围是 [1, 30000]。每个气温的值的都是 [30, 100] 范围内的整数。

思路：

可以正向遍历数组，也可以反向遍历数组，我这里采用的是正向遍历。

  1. 首先将第一个元素保存在一个容器中，然后遍历第二个元素；
  2. 然后遍历到第二个元素，将该元素与容器中的元素比较温度，如果当前元素温度比容器中的某个元素的温度高，则可以得到那个元素对应的答案（说明还要保存那个元素对应的索引值）；如果当前元素比容器中的所有元素的温度都低，那也把该元素放入容器中。
    可以发现： 只有当当前元素比容器中所有元素的温度都低时，才会保存在容器中，也就说明了该容器是递减的，可以只比较该容器最新进入的元素，如果比这个元素的值小，才需要比较更前面的元素，所以该容器为栈。

代码：
